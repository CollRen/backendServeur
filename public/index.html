Documentation de l'API

API donnant accès à une base de données Firebase contenant des informations sur des films

Routes pour les films

Ordonnés par
- titre, description, année, réalisation
Direction
- Ascendante (asc) ou descendante(desc)
Ajouter un nouveau film
- Objet dans le corps, aucun paramètre obligatoire
Limite
- Un nombre entier
GET /api/films
Retourne la liste de tous les films
GET /api/films/:id
Retourne le film ayant l'identifiant :id
POST /api/films
Retourne la liste de tous les films
GET /api/films
Ajoute un nouveau film à la liste de films

Le corps de la requête doit contenir les informations du film à ajouter. Retourne l'identifiant du film.

PUT /api/films/:id
Modifie le film ayant l'identifiant :id

Le corps de la requête doit contenir les informations du film à modifier. Le film sera modifié dans la base de données.

DELETE /api/films/:id
Supprime le film ayant l'identifiant :id de la base de données.

Routes pour les utilisateurs

Utilisateurs

POST /api/utilisateurs/inscription
Ajoute un nouvel utilisateur à la liste des utilisateurs

Le mot de passe sera encrypté avant d'être ajouté à la base de données. L'identifiant de l'utilisateur sera retourné. Notez que chaque adresse courriel est unique, si toutefois vous essayer d'utiliser une adresse courriel existante, un message d'erreur vous sera retourné.

POST /api/utilisateurs/connexion
Connecte un utilisateur

Le corps de la requête doit contenir les informations de l'utilisateur à connecter. Le mot de passe sera comparé avec celui dans la base de données. Si le mot de passe est valide, votre API retournera true sinon elle retournera false avec un message d'erreur approprié.
